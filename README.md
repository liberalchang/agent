# OpenClaw × RAGFlow 文档系统

本仓库提供一套用于工程化落地智能体（Agent）的**分层文档体系**，用于统一：

- 角色画像与行为边界
- 记忆沉淀与日志流水
- 工具与环境约束
- 引导启动与运行模式
- RAGFlow 检索增强治理规范

## 架构概览（先看这个再看细节）

### 每一层解决什么问题

- **身份层**：解决“我是谁/怎么说/为谁服务”，避免角色漂移。
- **行为层**：解决“能做什么/不能做什么/如何决策”，避免行为越界。
- **记忆层**：解决“哪些要长期记住/哪些只记当日”，避免记忆污染。
- **环境层**：解决“有什么工具/有什么约束/不可用如何降级”，避免能力幻觉。
- **引导层**：解决“启动先读什么/如何进入不同模式/如何检查检索可用性”，避免流程混乱。
- **治理层**：解决“怎么检索才一致/高频口径怎么统一/术语怎么对齐”，避免知识治理失序。

## 信息流（最小可用理解）

### 启动信息流

```
BOOTSTRAP
 → 加载身份层（SOUL → IDENTITY → USER）
 → 加载行为层（AGENTS → HEARTBEAT）
 → 加载记忆层（MEMORY → 当日日志）
 → 加载环境层（TOOLS）
 → 检查 RAGFlow 可用性
 → 建立当前任务上下文
 → 进入会话
```

### 问题处理信息流

```
用户问题
 → 判断任务类型
 → 检查是否需要检索增强
   → 若需要 → 调用 RAGFlow → 获取候选片段
   → 若不需要 → 直接进入回答生成
 → 结合身份/行为/记忆进行解释
 → 生成结构化输出
 → 判断是否需要沉淀记忆
 → 写入日志
```

## 快速开始

1. 按 `BOOTSTRAP.md` 的顺序装载文档上下文。
2. 处理任务时遵循 `AGENTS.md` 的决策优先级与边界规则。
3. 触发检索增强时遵循 `RAGFLOW_POLICY.md` 的检索流程与引用规范。
4. 需要长期保存的信息写入 `MEMORY.md`；当日过程信息写入 `memory/2026-03-23.md`。
